Sebastian 记忆系统架构图

基于 2026-04-18 新版 memory spec 生成 · 重点：artifact 协议、多后端存储、四 lane 检索、三类 consolidation

① 总体架构

首版只依赖数据库 + LLM。原始输入不会直接写入某张 memory 表，而是先变成标准化 artifacts，再进入 Normalize / Resolve / Persist 流程。

② 写入与冲突决策

LLM 只负责语义提炼，不直接控制数据库状态。最终写入动作统一由 Resolve 层根据 slot、cardinality 和 resolution policy 决定。

③ 读取与注入

读取不做“统一 top-k 塞 prompt”，而是先做意图判断，再按四条 lane 检索，最后在 Assembler 中做过滤和分区注入。

④ 后台沉淀与模型职责

Extractor 和 Consolidator 分工明确。Extractor 面向高频提取，Consolidator 面向低频归纳。两者都必须输出结构化结果，且不能绕过 resolver。
